Fundamentos de C#

C# é uma linguagem moderna e poderosa. Vamos começar com o essencial para você poder construir suas aplicações.

1. Instalação e Ambiente

Para começar a programar em C#, você precisa do .NET SDK (Software Development Kit). Ele inclui o compilador e tudo o que é necessário para rodar suas aplicações.

  • Instalação: Baixe o .NET SDK no site oficial da Microsoft. A instalação é simples e direta.
  • IDEs (Ambiente de Desenvolvimento): Você precisa de um editor de código. Os mais comuns são:
    • Visual Studio: A IDE mais completa da Microsoft, ideal para projetos grandes.
    • Visual Studio Code (VS Code): Um editor leve, rápido e muito popular, com extensões que o tornam poderoso para o desenvolvimento C#.

2. Variáveis e Tipos de Dados

Variáveis são como "caixas" que guardam informações. Cada caixa tem um tipo, que define o que pode ser armazenado nela.

// Tipos de dados comuns em C#
string nome = "Maria";       // Texto
int idade = 30;              // Números inteiros
double altura = 1.75;        // Números com casas decimais
bool eEstudante = true;      // Verdadeiro ou Falso

3. Estruturas de Controle (if/else)

Permitem que seu programa tome decisões com base em condições.

if (idade >= 18)
{
    Console.WriteLine("É maior de idade.");
}
else
{
    Console.WriteLine("É menor de idade.");
}

Importante: Sempre use chaves `{}` para delimitar blocos de código, mesmo que haja apenas uma linha dentro do bloco.

4. Estruturas de Repetição (for/foreach/while/do)

São estruturas onde repetem um determinado bloco de código por 'n' vezes, geralmente esta estrutura vem acompanhada de um índice ou valor booleano para indicar a parada do código em execução.

Exemplo de um laço `for`:

for (int i = 0; i < 5; i++)
{
    Console.WriteLine($"Contagem: {i}");
}

Esse laço irá imprimir os números de 0 a 4 no console.

Exemplo de um laço `foreach`:

foreach (var item in colecao)
{
    Console.WriteLine($"Item: {item}");
}

Esse laço irá iterar sobre cada item em uma coleção (como um array ou lista) e imprimir seu valor.

Exemplo de um laço `while`:

int i = 0;
while (i < 5)
{
    Console.WriteLine($"Contagem: {i}");
    i++;
}

Este laço imprimirá "Contagem: 0" até "Contagem: 4" no console. É crucial incrementar a variável `i` dentro do laço para evitar um loop infinito.

Exemplo de um laço `do while`:

int i = 0;
do
{
    Console.WriteLine($"Contagem: {i}");
    i++;
} while (i < 5);

Este laço imprimirá "Contagem: 0" até "Contagem: 4" no console. Assim como no `while`, é crucial incrementar a variável `i` dentro do laço para evitar um loop infinito. A principal diferença é que o `do while` garante que o bloco de código seja executado pelo menos uma vez.

5. Classes e Objetos: A Base da Orientação a Objetos

Pense em uma Classe como a planta de uma casa: ela define a estrutura (quantos quartos, banheiros, etc.). Um Objeto é a casa construída a partir dessa planta. Você pode construir várias casas (objetos) a partir da mesma planta (classe).

Em programação, uma classe agrupa variáveis (propriedades) e funções (métodos) que descrevem algo. Por exemplo, uma classe `Usuario`:

Importante Diferenciar:

Classe é a definição, Objeto é a instância, Propriedades são as características do objeto (dados armazenados/contidos), Métodos são as ações que um objeto pode realizar.

// A "planta" do nosso usuário
public class Usuario
{
    // Propriedades (as características)
    public int Id { get; set; }
    public string Nome { get; set; }
    public string Email { get; set; }

    // Um método (uma ação que o usuário pode fazer)
    public void Apresentar()
    {
        Console.WriteLine($"Olá, meu nome é {Nome} e meu email é {Email}.");
    }
}

// Criando um objeto (a "casa construída") a partir da classe
Usuario novoUsuario = new Usuario();
novoUsuario.Id = 1;
novoUsuario.Nome = "Ana";
novoUsuario.Email = "ana@email.com";

novoUsuario.Apresentar(); // Saída: Olá, meu nome é Ana e meu email é ana@email.com.

Relação com o Banco de Dados: Geralmente, uma classe como `Usuario` representa uma tabela no banco de dados. Cada objeto (`novoUsuario`) corresponde a uma linha nessa tabela, e cada propriedade (`Id`, `Nome`) corresponde a uma coluna.

6. Modificadores de Acesso

Modificadores de acesso controlam onde seu código pode ser acessado. Os mais comuns são:

  • public: Acessível de qualquer lugar.
  • private: Acessível apenas dentro da própria classe.
  • protected: Acessível dentro da própria classe e por classes que herdam dela.
  • internal: Acessível em qualquer lugar dentro do mesmo assembly (projeto).
public class ExemploAcesso
{
    public string Publico = "Eu sou público!";
    private string Privado = "Eu sou privado!";

    public void MostrarPrivado()
    {
        Console.WriteLine(Privado); // Pode acessar aqui
    }
}

// Fora da classe
ExemploAcesso obj = new ExemploAcesso();
Console.WriteLine(obj.Publico); // Acessível
// Console.WriteLine(obj.Privado); // ERRO: Não acessível

Usar o modificador de acesso correto é fundamental para a segurança e organização do seu código.

Fundamentos de MySQL

MySQL é o sistema que vai armazenar e gerenciar os dados da sua aplicação.

1. O que é um SGBD?

MySQL é um SGBD (Sistema de Gerenciamento de Banco de Dados). Pense nele como um "organizador" super eficiente para seus dados. Suas principais funções são:

  • Armazenar dados de forma segura e estruturada (em tabelas).
  • Recuperar dados rapidamente quando a aplicação solicita.
  • Garantir a consistência e a integridade dos dados (evitar informações duplicadas ou inválidas).
  • Controlar o acesso, definindo quem pode ver ou modificar quais informações.

2. Comandos SQL Essenciais

SQL é a linguagem que usamos para "conversar" com o SGBD. Aqui estão os comandos básicos para manipular uma tabela `usuarios`:

SELECT: Para buscar dados.

-- Busca todos os usuários da cidade de 'São Paulo'
SELECT nome, email FROM usuarios WHERE cidade = 'São Paulo';

INSERT: Para adicionar novos dados.

-- Adiciona um novo usuário à tabela
INSERT INTO usuarios (nome, email) VALUES ('Carlos', 'carlos@email.com');

UPDATE: Para atualizar dados existentes.

-- Atualiza o email do usuário 'Carlos'
UPDATE usuarios SET email = 'novo.email@email.com' WHERE nome = 'Carlos';

DELETE: Para remover dados.

-- Remove o usuário 'Carlos' da tabela
DELETE FROM usuarios WHERE nome = 'Carlos';

Conectando Tudo: API e Banco de Dados

A API (Application Programming Interface) é a ponte que conecta seu site (frontend) ao seu banco de dados (backend). Ela recebe pedidos do site, busca ou salva informações no banco de dados e devolve uma resposta.

Como funciona a comunicação?

Imagine que você quer ver a lista de todos os usuários cadastrados no seu site. O fluxo é o seguinte:

  1. Requisição (Frontend → API): Seu site (HTML/CSS/JS) envia uma requisição HTTP para um endereço da sua API, por exemplo, `GET /api/usuarios`.
  2. Processamento (API em C#): Sua API, escrita em C#, recebe essa requisição. O código C# entende que precisa buscar todos os usuários.
  3. Conexão com o BD (API → BD): A API usa uma biblioteca (como a `MySql.Data`) para abrir uma conexão com o banco de dados MySQL.
  4. Consulta SQL (API → BD): A API envia o comando `SELECT * FROM usuarios;` para o banco de dados.
  5. Execução (SGBD): O MySQL executa o comando, coleta todos os dados da tabela `usuarios` e os envia de volta para a API.
  6. Formatação (API em C#): A API recebe os dados brutos e os converte para um formato universal, geralmente JSON (JavaScript Object Notation).
  7. Resposta (API → Frontend): A API envia os dados em formato JSON de volta para o seu site como uma resposta à requisição inicial.
  8. Exibição (Frontend): Seu JavaScript recebe os dados JSON e os utiliza para exibir a lista de usuários na tela.

Frontend <-- Requisição HTTP --> API (C#) <-- Conexão/SQL --> Banco de Dados (MySQL)

Versionamento com Git e GitHub

Git e GitHub são ferramentas essenciais para qualquer desenvolvedor. Eles ajudam a controlar as versões do seu código e a colaborar com outras pessoas.

Git vs. GitHub

  • Git: É o sistema de controle de versão que roda na sua máquina. Ele salva "fotos" (commits) do seu projeto, permitindo que você volte no tempo se algo der errado.
  • GitHub: É uma plataforma online (um site) onde você pode hospedar seus projetos Git. É como uma "rede social para códigos", permitindo que você guarde uma cópia de segurança do seu projeto na nuvem e colabore com outros devs.

Instalação e Fluxo Básico

Primeiro, baixe e instale o Git. Depois, o fluxo para enviar seu projeto para o GitHub é:

# 1. Inicia um repositório Git na pasta do seu projeto
git init

# 2. Adiciona todos os arquivos para a próxima "foto" (commit)
git add .

# 3. Tira a "foto" com uma mensagem descritiva
git commit -m "Meu primeiro commit: criei a estrutura do projeto"

# 4. (No GitHub) Crie um novo repositório e copie a URL dele.
# 5. Conecte seu repositório local ao repositório remoto no GitHub
git remote add origin URL_DO_SEU_REPOSITORIO_NO_GITHUB

# 6. Envie suas alterações (commits) para o GitHub
git push -u origin main

Extra: GitHub Pages

O GitHub Pages é um recurso fantástico que permite hospedar sites estáticos (HTML, CSS, JS) de graça, diretamente do seu repositório. O site que você está vendo agora poderia ser hospedado com GitHub Pages! Você só precisa habilitar a opção nas configurações do seu repositório no GitHub.

Compêndio de Termos Técnicos

Um pequeno dicionário para ajudar a esclarecer alguns termos comuns que você encontrará em sua jornada.

API (Application Programming Interface)
Uma "ponte" que permite que diferentes sistemas de software se comuniquem entre si, trocando dados e funcionalidades de forma padronizada.
IDE (Integrated Development Environment)
Um programa que reúne várias ferramentas para facilitar a vida do programador, como editor de código, compilador e depurador. Ex: Visual Studio, VS Code.
JSON (JavaScript Object Notation)
Um formato de texto leve e de fácil leitura para troca de dados. É o padrão de comunicação em APIs web modernas.
SGBD (Sistema de Gerenciamento de Banco de Dados)
O software responsável por criar, manter e gerenciar um banco de dados, como o MySQL.
