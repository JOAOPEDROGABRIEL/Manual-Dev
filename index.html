Fundamentos de C#

C# é uma linguagem moderna e poderosa. Vamos começar com o essencial para você poder construir suas aplicações.

1. Instalação e Ambiente

Para começar a programar em C#, você precisa do .NET SDK (Software Development Kit). Ele inclui o compilador e tudo o que é necessário para rodar suas aplicações.

  • Instalação: Baixe o .NET SDK no site oficial da Microsoft. A instalação é simples e direta.
  • IDEs (Ambiente de Desenvolvimento): Você precisa de um editor de código. Os mais comuns são:
    • Visual Studio: A IDE mais completa da Microsoft, ideal para projetos grandes.
    • Visual Studio Code (VS Code): Um editor leve, rápido e muito popular, com extensões que o tornam poderoso para o desenvolvimento C#.

2. Variáveis e Tipos de Dados

Variáveis são como "caixas" que guardam informações. Cada caixa tem um tipo, que define o que pode ser armazenado nela.

// Tipos de dados comuns em C#
string nome = "Maria";       // Texto
int idade = 30;              // Números inteiros
double altura = 1.75;        // Números com casas decimais
bool eEstudante = true;      // Verdadeiro ou Falso

3. Estruturas de Controle (if/else)

Permitem que seu programa tome decisões com base em condições.

if (idade >= 18)
{
    Console.WriteLine("É maior de idade.");
}
else
{
    Console.WriteLine("É menor de idade.");
}

Importante: Sempre use chaves `{}` para delimitar blocos de código, mesmo que haja apenas uma linha dentro do bloco.

4. Estruturas de Repetição (for/foreach/while/do)

São estruturas onde repetem um determinado bloco de código por 'n' vezes, geralmente esta estrutura vem acompanhada de um índice ou valor booleano para indicar a parada do código em execução.

Exemplo de um laço `for`:

for (int i = 0; i < 5; i++)
{
    Console.WriteLine($"Contagem: {i}");
}

Esse laço irá imprimir os números de 0 a 4 no console.

Exemplo de um laço `foreach`:

foreach (var item in colecao)
{
    Console.WriteLine($"Item: {item}");
}

Esse laço irá iterar sobre cada item em uma coleção (como um array ou lista) e imprimir seu valor.

Exemplo de um laço `while`:

int i = 0;
while (i < 5)
{
    Console.WriteLine($"Contagem: {i}");
    i++;
}

Este laço imprimirá "Contagem: 0" até "Contagem: 4" no console. É crucial incrementar a variável `i` dentro do laço para evitar um loop infinito.

Exemplo de um laço `do while`:

int i = 0;
do
{
    Console.WriteLine($"Contagem: {i}");
    i++;
} while (i < 5);

Este laço imprimirá "Contagem: 0" até "Contagem: 4" no console. Assim como no `while`, é crucial incrementar a variável `i` dentro do laço para evitar um loop infinito. A principal diferença é que o `do while` garante que o bloco de código seja executado pelo menos uma vez.

5. Classes e Objetos: A Base da Orientação a Objetos

Pense em uma Classe como a planta de uma casa: ela define a estrutura (quantos quartos, banheiros, etc.). Um Objeto é a casa construída a partir dessa planta. Você pode construir várias casas (objetos) a partir da mesma planta (classe).

Em programação, uma classe agrupa variáveis (propriedades) e funções (métodos) que descrevem algo. Por exemplo, uma classe `Usuario`:

Importante Diferenciar:

Classe é a definição, Objeto é a instância, Propriedades são as características do objeto (dados armazenados/contidos), Métodos são as ações que um objeto pode realizar.

// A "planta" do nosso usuário
public class Usuario
{
    // Propriedades (as características)
    public int Id { get; set; }
    public string Nome { get; set; }
    public string Email { get; set; }

    // Um método (uma ação que o usuário pode fazer)
    public void Apresentar()
    {
        Console.WriteLine($"Olá, meu nome é {Nome} e meu email é {Email}.");
    }
}

// Criando um objeto (a "casa construída") a partir da classe
Usuario novoUsuario = new Usuario();
novoUsuario.Id = 1;
novoUsuario.Nome = "Ana";
novoUsuario.Email = "ana@email.com";

novoUsuario.Apresentar(); // Saída: Olá, meu nome é Ana e meu email é ana@email.com.

Relação com o Banco de Dados: Geralmente, uma classe como `Usuario` representa uma tabela no banco de dados. Cada objeto (`novoUsuario`) corresponde a uma linha nessa tabela, e cada propriedade (`Id`, `Nome`) corresponde a uma coluna.

6. Modificadores de Acesso

Modificadores de acesso controlam onde seu código pode ser acessado. Os mais comuns são:

  • public: Acessível de qualquer lugar.
  • private: Acessível apenas dentro da própria classe.
  • protected: Acessível dentro da própria classe e por classes que herdam dela.
  • internal: Acessível em qualquer lugar dentro do mesmo assembly (projeto).
public class ExemploAcesso
{
    public string Publico = "Eu sou público!";
    private string Privado = "Eu sou privado!";

    public void MostrarPrivado()
    {
        Console.WriteLine(Privado); // Pode acessar aqui
    }
}

// Fora da classe
ExemploAcesso obj = new ExemploAcesso();
Console.WriteLine(obj.Publico); // Acessível
// Console.WriteLine(obj.Privado); // ERRO: Não acessível

Usar o modificador de acesso correto é fundamental para a segurança e organização do seu código.

Fundamentos de MySQL

MySQL é o sistema que vai armazenar e gerenciar os dados da sua aplicação.

1. O que é um Banco de Dados?

Um banco de dados é uma coleção organizada de informações estruturadas, armazenadas eletronicamente em um sistema de computador. Os bancos de dados são projetados para facilitar o acesso, gerenciamento e atualização dos dados.

2. Consultas com SQL

SQL (Structured Query Language) é a linguagem padrão para interagir com bancos de dados. Através de "queries" (consultas), podemos realizar diversas operações, como buscar, inserir, atualizar e excluir dados.

Exemplo de consulta SQL:

SELECT nome, email FROM usuarios WHERE idade > 25;

Esta consulta busca o nome e o email de todos os usuários na tabela "usuarios" que têm mais de 25 anos.

3. O que é um SGBD?

MySQL é um SGBD (Sistema de Gerenciamento de Banco de Dados). Pense nele como um "organizador" super eficiente para seus dados. Suas principais funções são:

  • Armazenar dados de forma segura e estruturada (em tabelas).
  • Recuperar dados rapidamente quando a aplicação solicita.
  • Garantir a consistência e a integridade dos dados (evitar informações duplicadas ou inválidas).
  • Controlar o acesso, definindo quem pode ver ou modificar quais informações.

4. Comandos SQL Essenciais

Aqui estão os comandos básicos para manipular uma tabela `usuarios`:

SELECT: Para buscar dados.

-- Busca todos os usuários da cidade de 'São Paulo'
SELECT nome, email FROM usuarios WHERE cidade = 'São Paulo';

INSERT: Para adicionar novos dados.

-- Adiciona um novo usuário à tabela
INSERT INTO usuarios (nome, email) VALUES ('Carlos', 'carlos@email.com');

UPDATE: Para atualizar dados existentes.

-- Atualiza o email do usuário 'Carlos'
UPDATE usuarios SET email = 'novo.email@email.com' WHERE nome = 'Carlos';

DELETE: Para remover dados.

-- Remove o usuário 'Carlos' da tabela
DELETE FROM usuarios WHERE nome = 'Carlos';

5. Instalação e Ferramentas

Para interagir com bancos de dados MySQL, você precisará de algumas ferramentas:

  • MySQL Server: O próprio sistema de banco de dados. Baixe e instale a versão Community Server no site oficial do MySQL.
  • MySQL Workbench: Uma ferramenta gráfica para gerenciar seus bancos de dados MySQL. É excelente para criar tabelas, executar consultas e visualizar seus dados. Baixe-o junto com o servidor ou separadamente no mesmo site.
  • XAMPP/WAMP/MAMP (Opcional): Se você planeja desenvolver em outras linguagens (como PHP) ou prefere um pacote completo que inclua Apache, MySQL e PHP, pode usar um desses pacotes. Eles simplificam a instalação de um ambiente de desenvolvimento local.

Conectando Tudo: API e Banco de Dados

A API (Application Programming Interface) é a ponte que conecta seu site (frontend) ao seu banco de dados (backend). Ela recebe pedidos do site, busca ou salva informações no banco de dados e devolve uma resposta.

Como funciona a comunicação?

Imagine que você quer ver a lista de todos os usuários cadastrados no seu site. O fluxo é o seguinte:

  1. Requisição (Frontend → API): Seu site (HTML/CSS/JS) envia uma requisição HTTP para um endereço da sua API, por exemplo, `GET /api/usuarios`.
  2. Processamento (API em C#): Sua API, escrita em C#, recebe essa requisição. O código C# entende que precisa buscar todos os usuários.
  3. Conexão com o BD (API → BD): A API usa uma biblioteca (como a `MySql.Data`) para abrir uma conexão com o banco de dados MySQL.
  4. Consulta SQL (API → BD): A API envia o comando `SELECT * FROM usuarios;` para o banco de dados.
  5. Execução (SGBD): O MySQL executa o comando, coleta todos os dados da tabela `usuarios` e os envia de volta para a API.
  6. Formatação (API em C#): A API recebe os dados brutos e os converte para um formato universal, geralmente JSON (JavaScript Object Notation).
  7. Resposta (API → Frontend): A API envia os dados em formato JSON de volta para o seu site como uma resposta à requisição inicial.
  8. Exibição (Frontend): Seu JavaScript recebe os dados JSON e os utiliza para exibir a lista de usuários na tela.

Frontend <-- Requisição HTTP --> API (C#) <-- Conexão/SQL --> Banco de Dados (MySQL)

Criando uma API do Zero com .NET CLI

O .NET CLI (Command Line Interface) é uma ferramenta poderosa para criar, construir e executar aplicações .NET a partir da linha de comando. Vamos criar uma API simples passo a passo:

IMPORTANTE:

  1. Não esquecer de abrir o vs code na pasta do projeto, ou o projeto será armazenado na pasta errada ou global do seu computador.
  2. Certifique-se de ter o .NET SDK instalado em sua máquina.

Passo a passo

  1. Abra o Terminal: No VS Code, abra o terminal (View > Terminal).
  2. Crie um Novo Projeto: Use o seguinte comando para criar um novo projeto de API Web: 
    dotnet new webapi -n MinhaPrimeiraAPI

    Isso criará uma pasta chamada MinhaPrimeiraAPI com a estrutura básica de um projeto de API.

  3. Navegue até a Pasta do Projeto: 
    cd MinhaPrimeiraAPI
  4. Execute o Projeto: 
    dotnet run

    Isso iniciará a API. Você verá uma mensagem indicando em qual endereço ela está rodando (geralmente http://localhost:5000 ou https://localhost:5001).

  5. Explore a API: Abra seu navegador e vá para https://localhost:5001/swagger (ou o endereço indicado no terminal). Você verá a interface do Swagger, que permite testar os endpoints da sua API.

Entendendo a Estrutura do Projeto

O .NET CLI cria uma estrutura de projeto padrão. Alguns arquivos importantes:

  • Program.cs: Ponto de entrada da aplicação. Configura o host da aplicação, os serviços e o pipeline de requisições HTTP.
  • Startup.cs (ou similar em .NET 6+): Define os serviços que a aplicação usa (como controllers, CORS, Swagger) e como as requisições HTTP são processadas.
  • Controllers Pasta: Contém os controllers, que são classes que lidam com as requisições HTTP. O exemplo padrão (WeatherForecastController) mostra como retornar dados simples.
  • appsettings.json: Arquivo de configuração onde você pode armazenar configurações da aplicação, como strings de conexão com o banco de dados.

Modificando o Controller

Vamos modificar o controller padrão para retornar uma lista de usuários:

  1. Crie uma Classe Usuario: Crie uma nova classe chamada Usuario.cs: 
    public class Usuario
{
    public int Id { get; set; }
    public string Nome { get; set; }
    public string Email { get; set; }
}
  2. Modifique o Controller: Altere o WeatherForecastController (ou crie um novo) para retornar uma lista de objetos Usuario: 
    using Microsoft.AspNetCore.Mvc;
using System.Collections.Generic;

[ApiController]
[Route("[controller]")]
public class UsuariosController : ControllerBase
{
    [HttpGet]
    public IEnumerable<Usuario> Get()
    {
        return new List<Usuario>
        {
            new Usuario { Id = 1, Nome = "Ana", Email = "ana@email.com" },
            new Usuario { Id = 2, Nome = "Carlos", Email = "carlos@email.com" }
        };
    }
}
  3. Teste a API: Execute dotnet run novamente e acesse https://localhost:5001/usuarios no seu navegador. Você verá a lista de usuários em formato JSON.

Este é apenas o começo. A partir daqui, você pode adicionar mais endpoints, conectar a um banco de dados MySQL e implementar lógica de negócios mais complexa.

Exemplo de Código: API Simples em C#

Vamos ver um exemplo simplificado de como seria um controlador de API em C# usando ASP.NET Core, que é um framework popular para construir APIs.

using Microsoft.AspNetCore.Mvc;
using MySql.Data.MySqlClient; // Necessário para interagir com MySQL
using System.Collections.Generic;

[ApiController]
[Route("[controller]")] // Define a rota base para este controlador (ex: /usuarios)
public class UsuariosController : ControllerBase
{
    private readonly string _connectionString = "Server=localhost;Database=meubanco;Uid=root;Pwd=suasenha;";

    // GET: /usuarios
    [HttpGet]
    public IActionResult Get()
    {
        List<object> usuarios = new List<object>();
        using (MySqlConnection conexao = new MySqlConnection(_connectionString))
        {
            conexao.Open();
            string query = "SELECT Id, Nome, Email FROM usuarios";
            MySqlCommand comando = new MySqlCommand(query, conexao);
            
            using (MySqlDataReader reader = comando.ExecuteReader())
            {
                while (reader.Read())
                {
                    usuarios.Add(new {
                        Id = reader.GetInt32("Id"),
                        Nome = reader.GetString("Nome"),
                        Email = reader.GetString("Email")
                    });
                }
            }
        }
        return Ok(usuarios); // Retorna a lista de usuários como JSON (código 200 OK)
    }

    // POST: /usuarios
    [HttpPost]
    public IActionResult Post([FromBody] Usuario novoUsuario)
    {
        using (MySqlConnection conexao = new MySqlConnection(_connectionString))
        {
            conexao.Open();
            string query = "INSERT INTO usuarios (Nome, Email) VALUES (@Nome, @Email)";
            MySqlCommand comando = new MySqlCommand(query, conexao);
            comando.Parameters.AddWithValue("@Nome", novoUsuario.Nome);
            comando.Parameters.AddWithValue("@Email", novoUsuario.Email);
            comando.ExecuteNonQuery();
        }
        return Ok(); // Retorna sucesso (código 200 OK)
    }
}

Observações:

  • A string de conexão (_connectionString) deve ser configurada com os dados do seu banco de dados MySQL.
  • É necessário instalar o pacote NuGet MySql.Data no seu projeto para que a conexão com o MySQL funcione. Você pode fazer isso via CLI: 
    dotnet add package MySql.Data
  • IActionResult é um tipo de retorno flexível que permite retornar diferentes tipos de respostas HTTP (Ok, NotFound, BadRequest, etc.).
  • [FromBody] indica que o objeto novoUsuario virá do corpo da requisição HTTP (geralmente em JSON).
  • @Nome e @Email são parâmetros de consulta. Usá-los previne ataques de injeção de SQL, tornando seu código mais seguro.
  • Para interagir com o banco de dados de forma mais eficiente e utilizando o padrão de projeto Repository, considere adicionar os seguintes pacotes ao seu projeto:
    • dotnet add package Microsoft.EntityFrameworkCore.Design -v 8.0.6
dotnet add package Microsoft.EntityFrameworkCore.SqlServer -v 8.0.6
dotnet add package Swashbuckle.AspNetCore -v 6.4.0
  • O pacote Microsoft.EntityFrameworkCore.Design é necessário para executar comandos do Entity Framework Core, como migrations.
  • O pacote Microsoft.EntityFrameworkCore.SqlServer é o provedor de banco de dados para SQL Server. Se estiver utilizando outro banco de dados, instale o pacote correspondente.
  • O pacote Swashbuckle.AspNetCore é utilizado para gerar a documentação da API no formato Swagger.

A Connection String: A Chave para o Banco de Dados

A connection string é como um "endereço" completo que sua API usa para encontrar e se conectar ao banco de dados. Ela contém informações cruciais, como:

  • Servidor: O endereço do servidor MySQL (geralmente localhost se estiver rodando na mesma máquina).
  • Banco de Dados: O nome do banco de dados que você quer usar.
  • Usuário: O nome de usuário com permissão para acessar o banco de dados.
  • Senha: A senha do usuário.

Exemplo de Connection String:

Server=localhost;Database=meubanco;Uid=root;Pwd=minhasenha;

Onde colocar a Connection String?

Geralmente, a connection string é armazenada no arquivo appsettings.json do seu projeto. Isso permite que você altere a string sem precisar recompilar o código.

{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "AllowedHosts": "*",
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Database=meubanco;Uid=root;Pwd=minhasenha;"
  }
}

Como usar a Connection String no código C#:

using Microsoft.Extensions.Configuration; // Importe este namespace

public class UsuariosController : ControllerBase
{
    private readonly string _connectionString;

    public UsuariosController(IConfiguration configuration)
    {
        _connectionString = configuration.GetConnectionString("DefaultConnection");
    }

    // Seu código para acessar o banco de dados aqui...
}

No exemplo acima, a connection string é lida do arquivo de configuração e armazenada na variável _connectionString. Agora você pode usá-la para abrir uma conexão com o banco de dados:

using MySql.Data.MySqlClient;

public IActionResult Get()
{
    using (MySqlConnection conexao = new MySqlConnection(_connectionString))
    {
        conexao.Open();
        // Seu código SQL aqui...
    }
}

Segurança:

NUNCA coloque sua connection string diretamente no código-fonte, especialmente se você estiver usando um repositório público como o GitHub. Use variáveis de ambiente ou arquivos de configuração para armazená-la de forma segura.

Versionamento com Git e GitHub

Git e GitHub são ferramentas essenciais para qualquer desenvolvedor. Eles ajudam a controlar as versões do seu código e a colaborar com outras pessoas.

Git vs. GitHub

  • Git: É o sistema de controle de versão que roda na sua máquina. Ele salva "fotos" (commits) do seu projeto, permitindo que você volte no tempo se algo der errado. É utilizado uma estrutura de pilha para versionar código.
  • GitHub: É uma plataforma online (um site) onde você pode hospedar seus projetos Git. É como uma "rede social para códigos", permitindo que você guarde uma cópia de segurança do seu projeto na nuvem e colabore com outros devs.

GitHub para Iniciantes: Do Zero ao Deploy

Se você nunca usou o GitHub antes, não se preocupe! Vamos começar do início:

  1. Criando uma Conta:

    Acesse o site do GitHub e crie uma conta gratuita. Você precisará de um e-mail válido e escolher um nome de usuário.

  2. Instalando o Git:

    Git é o sistema de controle de versão que o GitHub usa. Baixe e instale o Git no seu computador. Durante a instalação, aceite as opções padrão (você pode alterá-las depois, se precisar).

  3. Configurando o Git:

    Abra o terminal (ou Git Bash, se estiver no Windows) e configure seu nome e e-mail. Isso é importante para que seus commits sejam identificados:

    git config --global user.name "Seu Nome Completo"
git config --global user.email "seuemail@example.com"
  4. Criando um Repositório:

    Um repositório (ou "repo") é como uma pasta para o seu projeto. No GitHub, clique em "New" (ou no ícone "+", depois "New repository"). Dê um nome ao seu repositório e escolha se ele será público (visível para todos) ou privado (visível apenas para você e quem você convidar).

  5. Enviando seu Projeto para o GitHub:

    Agora, vamos enviar os arquivos do seu projeto para o GitHub. No terminal, navegue até a pasta do seu projeto (use o comando cd). Depois, siga estes passos:

    git init                  # Inicia o Git na sua pasta
git add .                   # Adiciona todos os arquivos
git commit -m "Primeiro commit"  # Salva as alterações com uma mensagem
git remote add origin URL_DO_SEU_REPOSITORIO  # Conecta ao GitHub
git push -u origin main     # Envia os arquivos

    Substitua URL_DO_SEU_REPOSITORIO pela URL que o GitHub te deu quando você criou o repositório. Na primeira vez que você usar o git push, o Git pedirá seu nome de usuário e senha do GitHub.

  6. GitHub Pages: Publicando seu Site Gratuitamente:

    O GitHub Pages permite que você publique sites estáticos (HTML, CSS, JavaScript) diretamente do seu repositório. Para fazer isso:

    1. No seu repositório, vá em "Settings" (Configurações).
    2. Procure pela seção "Pages" (geralmente na barra lateral esquerda).
    3. Em "Source" (Fonte), escolha a branch "main" e a pasta "/ (root)".
    4. Clique em "Save" (Salvar).

    O GitHub Pages vai construir seu site e te dar um endereço (algo como https://seunome.github.io/nome-do-repositorio). Pode levar alguns minutos para o site ficar online.

Fundamentos de HTML, CSS e JavaScript

HTML, CSS e JavaScript são as tecnologias fundamentais para construir páginas web interativas.

HTML: A Estrutura da Web

HTML (HyperText Markup Language) é a espinha dorsal de qualquer página web. Ele usa tags para estruturar o conteúdo. As tags são como "blocos de construção" que informam ao navegador como exibir o conteúdo.

Como funciona: O navegador lê o HTML de cima para baixo, interpretando cada tag. A maioria das tags vêm em pares: uma tag de abertura (<tag>) e uma tag de fechamento (</tag>). O conteúdo entre essas tags é o que será exibido.

Exemplo:

<h1>Meu Título</h1> <!-- Define um título de nível 1 -->
<p>Este é um parágrafo.</p> <!-- Define um parágrafo de texto -->
<img src="imagem.jpg" alt="Uma imagem"> <!-- Insere uma imagem -->

Neste exemplo, <h1> define um título principal, <p> cria um parágrafo, e <img> insere uma imagem. O atributo src na tag <img> especifica o caminho para o arquivo da imagem, e alt fornece um texto alternativo caso a imagem não possa ser carregada.

CSS: A Beleza da Web

CSS (Cascading Style Sheets) é responsável pela aparência visual do seu site. Ele permite que você controle cores, fontes, layouts e muito mais.

Como funciona: O CSS usa seletores para "encontrar" os elementos HTML que você deseja estilizar. Um seletor pode ser o nome de uma tag (como h1 ou p), uma classe (.minha-classe) ou um ID (#meu-id).

Tipos de Seletores:

  • Seletor de Elemento: Aplica estilos a todos os elementos daquele tipo (ex: h1, p, img).
  • Seletor de Classe: Aplica estilos a elementos com um atributo class específico (ex: .destaque).
  • Seletor de ID: Aplica estilos a um único elemento com um atributo id específico (ex: #cabecalho). IDs devem ser únicos em uma página.

Exemplo:

h1 {
  color: blue;           /* Define a cor do texto como azul */
  text-align: center;   /* Centraliza o texto */
}
.destaque {
  font-weight: bold;   /* Define o texto como negrito */
}
#cabecalho {
  background-color: #f0f0f0; /* Define a cor de fundo como cinza claro */
}

Neste exemplo, todos os títulos <h1> serão azuis e centralizados. Qualquer elemento com a classe destaque terá o texto em negrito, e o elemento com o ID cabecalho terá um fundo cinza claro.

JavaScript: A Interatividade da Web

JavaScript é a linguagem que dá vida ao seu site. Ele permite que você crie interações, manipule o conteúdo da página e se comunique com servidores.

JavaScript e C#: Uma Analogia: Se você já conhece C#, pode pensar em JavaScript como um "primo" com algumas diferenças importantes:

  • Variáveis: Em C#, você declara o tipo de uma variável (int, string, etc.). Em JavaScript, você usa let, const ou var. JavaScript é dinamicamente tipada, o que significa que o tipo é verificado em tempo de execução.
  • Tipos de Dados: JavaScript tem tipos como number, string, boolean, object, array, null e undefined.
  • Estruturas de Controle: As estruturas if/else, for, while são muito semelhantes em C# e JavaScript.
  • Funções: Em C#, você tem métodos dentro de classes. Em JavaScript, você tem funções que podem existir independentemente.
  • JSON: Ambos C# e JavaScript usam JSON para trocar dados. Em C#, você usa bibliotecas como System.Text.Json para serializar/desserializar objetos. Em JavaScript, você tem os métodos nativos JSON.stringify() e JSON.parse().

Exemplo:

// Variáveis
let nome = "Maria"; // String
const idade = 30;   // Number (inteiro)
let altura = 1.75;  // Number (ponto flutuante)
let estudante = true; // Boolean

// Estrutura de controle
if (idade >= 18) {
  console.log("É maior de idade.");
} else {
  console.log("É menor de idade.");
}

// Estrutura de repetição
for (let i = 0; i < 5; i++) {
  console.log("Contagem: " + i);
}

// Manipulando o DOM (HTML)
document.getElementById("meuBotao").addEventListener("click", function() {
  alert("Botão clicado!");
});

// JSON
let usuario = { nome: "Carlos", email: "carlos@email.com" };
let jsonUsuario = JSON.stringify(usuario); // Converte objeto para JSON
console.log(jsonUsuario);

let objUsuario = JSON.parse(jsonUsuario);   // Converte JSON para objeto
console.log(objUsuario.nome);

Com HTML, CSS e JavaScript, você pode criar sites completos e dinâmicos. HTML fornece a estrutura, CSS define a aparência, e JavaScript adiciona a interatividade.

Compêndio de Termos Técnicos

Um pequeno dicionário para ajudar a esclarecer alguns termos comuns que você encontrará em sua jornada.

API (Application Programming Interface)
Uma "ponte" que permite que diferentes sistemas de software se comuniquem entre si, trocando dados e funcionalidades de forma padronizada.
IDE (Integrated Development Environment)
Um programa que reúne várias ferramentas para facilitar a vida do programador, como editor de código, compilador e depurador. Ex: Visual Studio, VS Code.
JSON (JavaScript Object Notation)
Um formato de texto leve e de fácil leitura para troca de dados. É o padrão de comunicação em APIs web modernas.
SGBD (Sistema de Gerenciamento de Banco de Dados)
O software responsável por criar, manter e gerenciar um banco de dados, como o MySQL.
SQL (Structured Query Language)
A linguagem padrão para interagir com bancos de dados relacionais, permitindo criar, ler, atualizar e deletar dados.
Versionamento
O processo de gerenciar mudanças no código-fonte ao longo do tempo, permitindo rastrear alterações, reverter para versões anteriores e colaborar com outros desenvolvedores.
Git
Um sistema de controle de versão distribuído que permite rastrear alterações no código-fonte ao longo do tempo. É amplamente utilizado para colaboração em projetos de software.
GitHub
Uma plataforma online que hospeda repositórios Git, facilitando a colaboração entre desenvolvedores e o compartilhamento de código.
Repositório
Um espaço de armazenamento no GitHub onde você pode manter e gerenciar seu código-fonte, documentação e outros arquivos relacionados a um projeto.
Commit
Uma "foto" do estado atual do código-fonte em um determinado momento. Cada commit tem uma mensagem descritiva que explica as mudanças feitas.
Branch
Uma ramificação do código-fonte que permite trabalhar em novas funcionalidades ou correções de bugs de forma isolada. As branches podem ser mescladas (merged) de volta à branch principal quando estiverem prontas.
Deploy
O processo de disponibilizar uma aplicação ou site para uso público. Isso pode incluir a transferência de arquivos para um servidor, configuração de ambientes e testes finais.
